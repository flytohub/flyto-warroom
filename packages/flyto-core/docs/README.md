# Documentation

`docs/` contains product, architecture, recipe, and release documentation for
flyto-core. Treat these files as the public contract for community users and as
operator guidance for self-hosted or enterprise deployments.

## Keep In Sync

- Recipe docs must match packaged recipe names and required inputs.
- Architecture docs should describe the shared runtime path used by CLI, MCP,
  and recipe execution.
- Security notes should explain opt-in behavior for browser, network, database,
  shell, and AI modules.
- Release notes should mention operationally relevant behavior changes.
