| 同一產品概念分散在多檔 | 檔案 |
|---|---|
| **CTEM 群** | `ctem.ts` (26 funcs), `ctem-extras.ts` (4), `scoring.ts` (12), `upstreamData.ts` (7), `assetEvidence.ts` (7) → **56 functions 散 5 檔** |
| **Repo 操作 / Scanning** | `repos.ts` (21), `arch.ts` (10), `verify.ts` (5), `autofix.ts` (11), `ci.ts` (5) — repos 又混 scan 又混 fix-plan 又混 GitHub proxy |

```
├── pentest/     ← pentest.ts + visualSimilarity.ts

├── history/     ← history.ts + history-feed.ts（合併）
├── platform/    ← orgs.ts + apiKeys.ts + capabilities.ts + businessUnits.ts
└── ...
```

依 memory 紀錄推測：
- **2026-04**：correlate engine + Pulse + warroom 同時 ship → `compounds/warroom/` 當時是合理的 single home
